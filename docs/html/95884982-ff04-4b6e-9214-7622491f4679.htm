      Introduction

      DotNetExtras is a collection of .NET Core client libraries that implement general-purpose and specialized functions that are frequently needed but not available in the .NET Core framework or in other public libraries. While most of the DotNetExtras code was developed independently, some of the functionality has been adapted or inspired by various opnline sources (developer forums, blog posts, etc). The libraries are oganized around tasks and grouped under the top-level namespaces. Each library can be included in a project via a NuGet package (coming soon).

      DotNetExtras.Common library

      Implements general-purpose classes for common tasks such as retrieving error information from the immediate and inner exceptions, simple JSON serialization and deserialization, generation of tokenized strings from collections, deep object cloning and comparison, setting object properties and getting property values using compound (nested) property names, etc. This library is a dependency for other specialized DotNetExtras libraries.

      Can be used in unit tests for assertion operations and mocking configuration settings.

      DotNetExtras.Web library

      Implements REST API client operations such as automatic refresh of the access tokens, parsing of the common error structures implemented by different providers (Azure, Apigee, OAuth, SCIM, etc.).
